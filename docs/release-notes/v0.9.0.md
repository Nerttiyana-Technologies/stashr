# stashr v0.9.0 — first public release

**A fully .NET, open-source secrets engine.** Vault-compatible API, native .NET integration, and
a built-in web UI — all in one language you can read, debug, and contribute to.

> ⚠️ **Pre-audit release.** stashr `0.9.0` is feature-complete for near-Vault parity but has not
> yet had an independent third-party security audit. Use it for evaluation, internal tooling, and
> non-critical workloads — not crown-jewel production secrets — until the audit completes. `1.0.0`
> is reserved for the first post-audit release.

## Highlights

- **Native to .NET.** Engine, server, SDK, CLI, and UI are all .NET. Bind secrets straight into
  `IConfiguration` with `AddStashr()`, or — for .NET Framework 4.8 — resolve `web.config` values
  through a `ConfigurationBuilder`. No more plaintext in `appsettings.json` / `app.config`.
- **Near-Vault parity.** KV v2, Transit, dynamic database credentials, Cubbyhole, Identity,
  AppRole + token auth, deny-by-default policies, response wrapping, leases, and a
  Vault-compatible `/v1` API.
- **A differentiator: explain-access.** Ask the policy engine *why* a request would be allowed
  or denied — not just whether.
- **Operationally serious.** Shamir seal/unseal, pluggable auto-unseal, PostgreSQL-backed HA with
  fencing and fail-closed self-seal, TLS/mTLS, FIPS-aware crypto, secure memory, and a
  tamper-evident HMAC hash-chained audit log.
- **Built-in web UI.** A Blazor WebAssembly admin console at `/ui`, with light and dark themes,
  shipped inside the server binary.

## Get started in 5 minutes

```bash
docker compose up --build
# root token is printed in the logs; API on http://localhost:8080, UI at http://localhost:8080/ui
```

Or run from source with the .NET 10 SDK: `dotnet run --project src/Stashr.Server`.
Full instructions in [BUILDING.md](../../BUILDING.md).

## Install options

- **Docker image** (multi-stage, non-root) — plus single-node and two-node HA Compose stacks.
- **`dotnet` global tool** — `stashr` CLI.
- **Self-contained single binary** — no runtime needed on the host.

## Compatibility

- Server & engine: **.NET 10**.
- SDK, configuration provider, CLI consumers: **.NET 10 / .NET 8 / .NET Standard 2.0**.
- Legacy configuration builder: **.NET Framework 4.8**.
- Storage: **PostgreSQL** (production), in-memory (dev/test).

## Security & compliance

Read the precise posture in [COMPLIANCE.md](../../COMPLIANCE.md) and report issues per
[SECURITY.md](../../SECURITY.md). FIPS-validated cryptography is inherited from a FIPS-configured
host OS; stashr is not itself CMVP-validated, FedRAMP-authorized, or independently audited yet.
